# Reproducing GeoMM-Bench Results and Figures

Everything below runs on a normal machine with internet (model weights download
from Hugging Face on first use). Tested on Apple Silicon (MPS); CUDA/CPU also work.

## 1. Install
```bash
pip install -r requirements.txt
```

## 2. Reproduce the numbers

Text-only (no imagery; downloads CLIP weights on first run):
```bash
python scripts/run_evaluation.py --approaches text_only --out results/reproduced_results.json
```

All approaches (needs the source log PDF; see DATASHEET.md on availability):
```bash
python scripts/run_evaluation.py \
  --approaches text_only vision_clip fusion grounding_dino blip2 \
  --logs-pdf path/to/vilkyciai22_logs500.pdf \
  --out results/reproduced_results.json
```

Expected (pilot, n=11): text-only macro-F1 0.886 / accuracy 0.909;
vision-only CLIP 0.620; Grounding DINO 0.071 / 0.091; BLIP-2 accuracy 0.182.

## 3. Regenerate the figures (from the numbers)

```bash
python scripts/make_results_figure.py \
  --results results/reproduced_results.json \
  --out images/Figure2_Results_Comparison.png

python scripts/make_architecture_figure.py --out images/architecture.png
```

The results figure is plotted directly from the results JSON, so it cannot
diverge from the reported numbers. Where a value is not reported (e.g. BLIP-2
macro-F1), the bar is shown as `n/a` rather than filled with a placeholder.

## 4. One command

```bash
bash scripts/reproduce.sh                                   # text-only + figures
LOGS_PDF=path/to/vilkyciai22_logs500.pdf bash scripts/reproduce.sh full   # all + figures
```

## Note on the Vilkyciai data figure
`fig1_vilkyciai.png` (Figure 1 in the paper) is a rendering of the source log
and interpretation rasters, not an output of the evaluation. It is produced
separately from the operator PDFs and is not regenerated by this pipeline.
